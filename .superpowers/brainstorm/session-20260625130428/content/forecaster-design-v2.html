Forecaster 设计（修订版）

纯时序预测 — 无节假日/促销注入，仅基于历史模式

数据现状

数据源RetailRocket（已脱敏）
事件类型view / addtocart / transaction
时间粒度毫秒级 Unix timestamp → 需聚合到日/周
节假日无（脱敏后不可知）
促销事件无
外部回归量无

预测目标（按日聚合后）

日交易量 / GMV

按日统计 transaction 数量或金额（如无金额则用订单数代替）

日 PV / UV

view 事件日活，预测流量趋势

日转化率

transaction / visitor 日转化率曲线

Prophet 参数策略

无外部回归量，只有趋势+季节性

model = Prophet(
    yearly_seasonality='auto',     # 数据不足1年自动关闭
    weekly_seasonality=True,       # 电商周内规律明显
    daily_seasonality=False,       # 日聚合后无日内周期
    changepoint_prior_scale=0.05,  # 默认，不过度拟合突变点
    seasonality_mode='additive'    # 加法模型，GMV天然可加
)

核心 API（精简后）

Forecaster 类 — 去掉 holidays / events / 多品类
class Forecaster:
    def __init__(self, df, date_col='ds', target_col='y',
                 freq='D',
                 changepoint_prior_scale=0.05,
                 seasonality_mode='additive')

    def fit(self)           → self
    def predict(self, periods=30)  → DataFrame [ds, yhat, yhat_lower, yhat_upper]
    def plot(self)           → Plotly Figure
    def components(self)     → 趋势/周季节性分解图
    def metrics(self)        → MAE / RMSE / MAPE（回测）

与 Anomaly Detector 集成

A

残差检测（推荐）

fit 后对历史数据做 predict，计算实际值与预测值的残差。残差超出历史 3σ 范围的点标记为异常。不改现有 3σ+IQR 逻辑，作为新增检测层。

B

置信区间检测

观测值落在 yhat_lower 以下或 yhat_upper 以上 → 异常。对突变点更敏感但可能有更多误报。
